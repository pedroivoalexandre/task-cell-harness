# Prompt Templates

## Purpose

This document defines the compact prompts used by the simulated Task Cell Harness workflow.
The templates are intentionally short so they remain easy to review and adapt.

## General Rules

- keep context minimal;
- state the acceptance criteria explicitly;
- list forbidden actions clearly;
- do not request real-agent execution;
- do not assume network access;
- keep output structured and deterministic when possible.

## Template Types

- implementer prompt;
- reviewer prompt;
- consolidated report prompt.
